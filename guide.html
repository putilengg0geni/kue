02
üñ•Ô∏è PROSEDUR SMARTPLS 4 Modeling Canvas
Membuat Measurement Model (Outer Model)
  1. Klik tombol "Modeling".
  2. Pilih indikator TL1 s.d TL11 dari panel kiri.
  3. Drag & drop ke canvas > Rename jadi "TL".
  4. Ulangi untuk OI (OI1-OI5), MI (MI1-MI5), dan UPB (UPB1-UPB6).
‚öôÔ∏è SCRIPT R (analysis_sempls.R) functions: constructs()
Definisi Outer Model 
measurements <- constructs(
  composite("TL", multi_items("TL", 1:11)),
  composite("OI", multi_items("OI", 1:5)),
  composite("MI", multi_items("MI", 1:5)),
  composite("UPB", multi_items("UPB", 1:6)),
  ...
)
‚úÖ KESIMPULAN STEP 2: R mendefinisikan variabel laten sama persis dengan cara drag-drop di SmartPLS.
03
üñ•Ô∏è PROSEDUR SMARTPLS 4 Connecting Arrows
Membuat Structural Model (Jalur Panah)
  1. Klik tools "Connect" (Panah).
  2. Tarik garis dari TL ‚Üí OI.
  3. Tarik garis dari TL ‚Üí UPB.
  4. Tarik garis dari OI ‚Üí UPB.
  5. Tarik garis MI ‚Üí UPB (Wajib untuk perhitungan moderasi).
‚öôÔ∏è SCRIPT R (analysis_sempls.R) functions: relationships() 
structure <- relationships(
  paths(from = "TL", to = c("OI", "UPB")),
  paths(from = "OI", to = "UPB"),
  # Main effect of Moderator (Required)
  paths(from = "MI", to = "UPB"), 
  ...
)
‚úÖ KESIMPULAN STEP 3: Struktur jalur (path) identik.
04
üñ•Ô∏è PROSEDUR SMARTPLS 4 Moderating Effect
Menambahkan Efek Moderasi (H5)
  1. Klik kanan pada UPB (Dependent).
  2. Pilih Add Moderating Effect.
  3. Setting pop-up:
    • Moderator: MI
    • Independent: OI
    • Method: Two Stage (Default)
  4. Klik OK ‚Üí Muncul lingkaran hijau baru ke arah UPB.
‚öôÔ∏è SCRIPT R (analysis_sempls.R) function: interaction_term() 
# Di dalam measurements:
interaction_term(iv = "OI", moderator = "MI", 
                 method = two_stage)

# Di dalam structure:
paths(from = "OI*MI", to = "UPB")

Note: "OI*MI" adalah nama otomatis untuk variabel interaksi.

‚úÖ KESIMPULAN STEP 4: Inilah kunci "Sengketa Gambar". Baik SmartPLS maupun R secara statistik menghitung "Interaksi" sebagai variabel baru.
05
üñ•Ô∏è PROSEDUR SMARTPLS 4 Running
Eksekusi & Bootstrapping
  1. Menu Calculate > PLS-SEM Algorithm (untuk Cari R¬≤ & Path Coef).
  2. Menu Calculate > Bootstrapping (untuk Cari Signifikansi/P-Value).
  3. Setting: Subsamples = 500.
  4. Start Calculation.
‚öôÔ∏è SCRIPT R (analysis_sempls.R) estimate & bootstrap 
# 1. PLS Algorithm
pls_model <- estimate_pls(
  data = analysis_data, 
  measurement_model = measurements, 
  structural_model = structure
)

# 2. Bootstrapping
boot_model <- bootstrap_model(
  pls_model, 
  nboot = 500
)
